# AI Learning System: NotebookLM + Gemini + Obsidian

> [!summary] Что важного в видео
> Идея не в том, чтобы просто быстрее читать. Идея в том, чтобы пропускать входящий материал через фильтр: убрать воду, сравнить с уже известным, оставить только новые полезные знания и превратить их в заметки, карточки, тесты и план действий.

## Инструменты из видео

| Инструмент | Для чего использовать |
| --- | --- |
| NotebookLM | Загружать источники: видео, статьи, PDF, документацию, книги. Получать выжимку только по этим источникам. |
| Gemini | Делать преобразования: сравнение, план, таблицы, инфографика, тесты, карточки, проектные идеи. |
| Obsidian | Хранить свою базу знаний, контекст, связи, ошибки, команды, прогресс. |
| AI agent | Работать с твоими локальными файлами: искать дубли, обновлять заметки, делать шаблоны, связывать темы. |

## Как применить в нашей связке

> [!workflow] Рабочий цикл
> 1. Кидаешь материал в NotebookLM.
> 2. Просишь выжимку: факты, команды, термины, ошибки, что реально применить.
> 3. Даёшь мне ссылку/текст/выжимку и говоришь: `сравни с моим Obsidian vault`.
> 4. Я ищу по заметкам, что уже есть, что новое, где дубли.
> 5. Я создаю или обновляю заметки в правильных папках.
> 6. В конце делаем карточки, задания, чеклист и следующий шаг.

## Команды для меня как AI агента

```text
Изучи этот материал и сравни с моим vault DevOps-Linux. Не добавляй дубли, выдели только новое.
```

```text
Сделай из этого конспект: кратко, команды отдельно, ошибки отдельно, практические задания отдельно.
```

```text
Создай карточки повторения по этой теме и добавь ссылки на связанные заметки.
```

```text
Проверь мои заметки по теме Docker/Linux/Git и скажи, чего не хватает для практики.
```

## Фильтр нового материала

Перед добавлением в Obsidian проверяем:

1. Это уже есть в базе?
2. Это можно применить в твоём DevOps/Linux обучении?
3. Есть ли команда, пример, ошибка или упражнение?
4. Куда это положить: `00_Inbox`, `02_Linux`, `04_DevOps`, `06_Errors`, `07_Cheatsheets`, `08_Git`?
5. Нужно ли сделать карточку или задачу?

## Формат хорошей заметки

> [!ai] Минимальный стандарт
> Хорошая учебная заметка должна отвечать на четыре вопроса: что это, зачем нужно, как использовать, какие ошибки бывают.

```markdown
# Тема

## Коротко

## Когда использовать

## Команды / примеры

## Частые ошибки

## Практика

## Связанные заметки
```

## Как использовать NotebookLM без потери качества

1. Не просить “объясни всё”. Просить “выдели только уникальные идеи и практические шаги”.
2. Не копировать весь ответ в Obsidian. Сначала фильтровать.
3. Отдельно просить тесты и карточки.
4. Отдельно просить список неизвестных терминов.
5. Отдавать мне результат для интеграции в локальную базу.
