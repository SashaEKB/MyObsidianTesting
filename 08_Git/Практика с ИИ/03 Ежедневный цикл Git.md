# Ежедневный цикл Git

Это главный цикл, который используется постоянно.

Цепочка: изменил файл -> `git status` -> `git add` -> `git commit` -> `git push`.

Коротко:

- `git status` — посмотреть, что изменилось.
- `git add` — подготовить изменения.
- `git commit` — сохранить изменения в историю.
- `git push` — отправить commits на GitHub.

## Посмотреть состояние

```bash
git status
```

Показывает:

- новые файлы;
- измененные файлы;
- удаленные файлы;
- что уже готово к коммиту.

## Подготовить один файл

```bash
git add hello.txt
```

`git add` добавляет файл в подготовку к коммиту.

Важно: это еще не сохранение в историю.

## Подготовить все изменения

```bash
git add .
```

Точка `.` означает текущую папку и все внутри нее.

Используй осторожно: команда добавит все изменения сразу.

## Создать коммит

```bash
git commit -m "add hello file"
```

`git commit` сохраняет подготовленные изменения в историю.

`-m` добавляет короткое сообщение коммита.

## Хорошее сообщение коммита

Пиши коротко: что было сделано.

Примеры:

```text
add hello file
update git notes
fix typo in readme
```
