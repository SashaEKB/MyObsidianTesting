# Ошибки новичка

## Author identity unknown

Git не знает имя и email автора коммитов.

Решение:

```bash
git config --global user.name "Kraz"
git config --global user.email "your-email@example.com"
```

## nothing to commit, working tree clean

Это не ошибка.

Git говорит: изменений нет, все сохранено.

## Password authentication is not supported

Ошибка при `git push` через HTTPS:

```text
Password authentication is not supported for Git operations.
```

GitHub больше не принимает обычный пароль.

Решения:

- использовать Personal Access Token вместо пароля;
- или настроить SSH-ключи.

## GitHub снова просит логин и токен

Если remote начинается с `https://`, GitHub работает через HTTPS.

Проверить:

```bash
git remote -v
```

Если хочешь использовать SSH, ссылка должна быть такой:

```text
git@github.com:SashaEKB/testing-GIT.git
```

Поменять HTTPS на SSH:

```bash
git remote set-url origin git@github.com:SashaEKB/testing-GIT.git
```

## ssh -T git@github.com зависает

Если команда зависла на строке:

```text
Connecting to github.com ... port 22
```

Скорее всего, порт `22` не отвечает или блокируется.

Решение: использовать SSH через порт `443`.

Создать или изменить файл:

```text
~/.ssh/config
```

Добавить:

```sshconfig
Host github.com
  HostName ssh.github.com
  User git
  Port 443
  IdentityFile ~/.ssh/id_ed25519
```

После этого снова проверить:

```bash
ssh -T git@github.com
```

## GitHub пишет successfully authenticated

Сообщение:

```text
Hi SashaEKB! You've successfully authenticated, but GitHub does not provide shell access.
```

Это хороший результат.

GitHub говорит: ключ принят, но обычный SSH-доступ к серверу не выдается.

Для `git push` и `git pull` этого достаточно.

## Everything up-to-date

Сообщение после `git push`:

```text
Everything up-to-date
```

Это значит: отправлять нечего, GitHub уже содержит твои последние коммиты.

## The current branch main has no upstream branch

Ошибка после `git push`:

```text
fatal: The current branch main has no upstream branch.
```

Смысл: Git знает адрес удаленного репозитория `origin`, но текущая локальная ветка `main` еще не связана с веткой `main` на GitHub.

Так часто бывает, когда:

- создал новый repository на GitHub;
- поменял `origin` на другой repository;
- первый раз отправляешь новую branch.

Решение:

```bash
git push -u origin main
```

`-u` связывает локальную ветку `main` с удаленной веткой `origin/main`.

После этого можно писать просто:

```bash
git push
```

Проверить связь:

```bash
git branch -vv
```

## Не сделал git add перед commit

Если выполнить `git commit`, но ничего не добавлено через `git add`, Git не создаст коммит.

Проверь:

```bash
git status
```

Потом добавь изменения:

```bash
git add .
```

## Вышел из папки проекта

Если Git пишет, что это не репозиторий, возможно ты не в той папке.

Проверь текущую папку:

```bash
pwd
```

`pwd` показывает полный путь текущей папки.

## Не бойся git status

Если не понимаешь, что происходит, первая команда почти всегда:

```bash
git status
```
